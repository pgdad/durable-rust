# Python/Rust Compliance Suite

Verifies behavioral parity between the Python durable Lambda SDK and this Rust implementation by comparing operation sequences from identical workflows.

## How It Works

1. **Python reference workflows** (`python/workflows/`) implement business logic using the Python durable Lambda SDK
2. **Rust implementations** (`rust/src/`) implement the same workflows using `durable-lambda-closure`
3. **JSON fixtures** (`tests/fixtures/`) store the expected operation sequences generated by the Python workflows
4. **Comparison tests** (`rust/tests/`) run each Rust workflow and verify it produces the same operation sequence as the Python reference

Python is **not required in CI**. The Python workflows are run manually to generate JSON fixtures, which are committed to the repo. CI only runs Rust tests against those fixtures.

## Running the Suite

### Run Rust compliance tests (CI)

```sh
cargo test -p durable-lambda-compliance
```

### Regenerate Python fixtures (manual)

```sh
cd compliance/python
pip install -r requirements.txt
python workflows/order_processing.py > ../tests/fixtures/order_processing.json
python workflows/parallel_fanout.py > ../tests/fixtures/parallel_fanout.json
python workflows/callback_approval.py > ../tests/fixtures/callback_approval.json
```

## Adding a New Compliance Workflow

1. Create a Python reference implementation in `python/workflows/new_workflow.py` that outputs a JSON operation sequence
2. Create a matching Rust implementation in `rust/src/new_workflow.rs`
3. Add the module to `rust/src/lib.rs`
4. Generate the fixture: `python workflows/new_workflow.py > ../tests/fixtures/new_workflow.json`
5. Add a comparison test in `rust/tests/compare_outputs.rs`

## Operation Sequence Format

Each workflow produces a JSON operation sequence for comparison:

```json
{
  "workflow": "order_processing",
  "operations": [
    {"type": "step", "name": "validate_order"},
    {"type": "step", "name": "charge_payment"},
    {"type": "step", "name": "send_confirmation"}
  ]
}
```

The comparison checks:
- Operation count matches exactly
- Each operation's type and name match in order
- Any divergence produces a clear diff in the test failure output
