# Macro-Style Examples

The macro-style API uses the `#[durable_execution]` proc-macro to generate the Lambda runtime boilerplate and `main()` function. You write only the handler logic.

**When to choose this style:** You want zero boilerplate and minimal ceremony. The macro handles runtime setup, so your code is just the handler function.

## Quick Start

```bash
cargo build -p macro-style-example
```

## Handler Pattern

```rust
use durable_lambda_core::context::DurableContext;
use durable_lambda_core::error::DurableError;
use durable_lambda_macro::durable_execution;

#[durable_execution]
async fn handler(
    event: serde_json::Value,
    mut ctx: DurableContext,
) -> Result<serde_json::Value, DurableError> {
    let result: Result<String, String> = ctx.step("work", || async {
        Ok("done".to_string())
    }).await?;
    Ok(serde_json::json!({"result": result.unwrap()}))
}
// main() is generated by the #[durable_execution] macro
```

## Handlers

This crate contains 11 example handlers covering all core durable operations.

| Binary | Source | Description |
|--------|--------|-------------|
| `macro-basic-steps` | `src/basic_steps.rs` | Checkpointed work units -- extract and validate data across steps |
| `macro-step-retries` | `src/step_retries.rs` | Automatic retry with exponential backoff on transient failures |
| `macro-typed-errors` | `src/typed_errors.rs` | Custom error types with serde serialization through step results |
| `macro-parallel` | `src/parallel.rs` | Concurrent fan-out with independent branches using `ctx.parallel()` |
| `macro-map` | `src/map.rs` | Parallel collection processing with batching via `ctx.map()` |
| `macro-child-contexts` | `src/child_contexts.rs` | Isolated subflows with independent checkpoint namespaces |
| `macro-replay-safe-logging` | `src/replay_safe_logging.rs` | Structured logging that is suppressed during replay |
| `macro-combined-workflow` | `src/combined_workflow.rs` | Multi-operation workflow combining steps, waits, and parallel execution |
| `macro-callbacks` | `src/callbacks.rs` | External signal coordination -- suspend and resume on external events |
| `macro-waits` | `src/waits.rs` | Time-based suspension with `ctx.wait()` |
| `macro-invoke` | `src/invoke.rs` | Durable Lambda-to-Lambda invocation via `ctx.invoke()` |

## Running

These are AWS Lambda handlers and cannot be run locally with `cargo run`. To execute them:

- **Deploy to AWS Lambda** with durable execution enabled, or
- **Write tests** using `MockDurableContext` from the `durable-lambda-testing` crate (no AWS credentials needed)

See the [root README](../../README.md) for full API documentation, testing patterns, and deployment instructions.
