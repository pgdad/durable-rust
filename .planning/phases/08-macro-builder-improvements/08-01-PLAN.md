    - Test: existing valid handler with correct signature still passes validation
  </behavior>
  <action>
    Extend `validate_signature()` in `expand.rs` with two new checks after the existing async and param-count checks:

    1. **Second parameter type check (FEAT-29):**
       - Extract second param via `func.sig.inputs.iter().nth(1)` (safe — param count already validated as 2)
       - Match `FnArg::Typed(PatType { ty, .. })` then check `ty` is `Type::Path` whose last path segment ident is `"DurableContext"`
       - If not, return `Error::new_spanned(ty, "...")` with message: `#[durable_execution] second parameter must be DurableContext, found \`{ident}\` — expected: async fn handler(event: serde_json::Value, ctx: DurableContext) -> Result<serde_json::Value, DurableError>`
       - If the type is not a `Type::Path` at all (e.g. reference type), also reject with a message mentioning DurableContext
       - Use `if let FnArg::Typed(PatType { ty, .. }) = second` — defensive against FnArg::Receiver even though free functions don't have it

    2. **Return type check (FEAT-30):**
       - Match `func.sig.output` against `ReturnType::Type(_, boxed_type)` and `ReturnType::Default`
       - For `ReturnType::Default`: return `Error::new_spanned(&func.sig.fn_token, "#[durable_execution] must explicitly return Result<serde_json::Value, DurableError>")`
       - For `ReturnType::Type(_, boxed_type)`: check `boxed_type` is `Type::Path` whose last segment ident is `"Result"`
       - If not Result: return `Error::new_spanned(boxed_type, "#[durable_execution] return type must be Result<serde_json::Value, DurableError> — found non-Result type")`

    3. **Add new unit tests** in the existing `#[cfg(test)] mod tests` block:
       - `rejects_wrong_second_param_type`: parse_quote a function with `(x: i32, y: i32)` params, verify error contains "DurableContext"
       - `rejects_non_result_return_type`: parse_quote a function with `-> String`, verify error contains "Result"
       - `rejects_missing_return_type`: parse_quote a function with no return type, verify error contains "Result"
       - `accepts_mut_binding_on_context`: parse_quote with `mut ctx: DurableContext`, verify expansion succeeds (this is the existing test, just confirm it still passes)

    Add `use syn::{FnArg, PatType, ReturnType, Type};` to the imports (alongside existing `use syn::{Error, ItemFn};`).
