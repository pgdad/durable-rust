<task type="auto">
  <name>Task 1: Create build-images.sh with parallel crate builds and ECR push</name>
  <files>scripts/build-images.sh</files>
  <action>
Create `scripts/build-images.sh` (executable, `chmod +x`) following the pattern from `scripts/deploy-ecr.sh`. The script must:

**Header and variables:**
- `#!/usr/bin/env bash` with `set -euo pipefail`
- `SCRIPT_DIR`, `REPO_ROOT`, `TF_DIR` computed from script location (same pattern as deploy-ecr.sh)
- `PROFILE="adfs"`, `REGION="us-east-2"` constants

**Prerequisites gate:**
- Call `"$SCRIPT_DIR/verify-prerequisites.sh" || exit 1`

**ECR login (once, before all builds):**
- `ECR_URL=$(terraform -chdir="$TF_DIR" output -raw ecr_repo_url)` -- never hardcode
- `aws ecr get-login-password --profile "$PROFILE" --region "$REGION" | docker login --username AWS --password-stdin "$ECR_URL"`

**Base image pre-pull (before parallelism):**
- `docker pull lukemathwalker/cargo-chef:latest-rust-1`
- `docker pull public.ecr.aws/lambda/provided:al2023`
This prevents Docker layer-store contention when 4 parallel builds all try to pull the same base images simultaneously (Research Pitfall 3).

**Crate-to-binary mapping:**
Define an associative array `CRATE_BINS` mapping each of the 4 crate names to its 11 space-separated binary names. All 44 binary names listed explicitly (from the Complete Binary Manifest in the research doc). Do NOT compute binary names dynamically -- hardcode them to match lambda.tf exactly.

**Build function `build_and_push_crate()`:**
Takes two args: `$1` = package name (crate), `$2` = space-separated binary names.
For each binary in the list:
1. `docker build -f "$REPO_ROOT/examples/Dockerfile" --build-arg "PACKAGE=${package}" --build-arg "BINARY_NAME=${bin_name}" -t "${ECR_URL}:${bin_name}" "$REPO_ROOT"`
2. `docker push "${ECR_URL}:${bin_name}"`
3. Log progress: `echo "[${package}] Pushed ${bin_name} (N/11)"`
After all 11: `echo "[${package}] Complete -- 11 images pushed"`

**Parallel execution:**
Launch 4 background jobs (one per crate) using `&`, collect PIDs in an array. Wait for each PID individually, counting failures. Exit 1 if any failed (Research Pitfall 5 pattern).

**Final summary:**
Print total images pushed and ECR URL. Run a quick `aws ecr list-images` count to verify 44 tags exist:
```bash
IMAGE_COUNT=$(aws ecr list-images --profile "$PROFILE" --region "$REGION" \
  --repository-name "$(basename "$ECR_URL")" \
  --query 'length(imageIds)' --output text)
echo "ECR image count: $IMAGE_COUNT (expected: 44)"
```

**Output prefixing:** Let the function itself prefix its own output (echo statements already include `[${package}]` prefix), and run `build_and_push_crate "$package" "${CRATE_BINS[$package]}" &` directly. This preserves the PID exit code for proper error collection. Do NOT pipe through sed as it loses the exit code.
