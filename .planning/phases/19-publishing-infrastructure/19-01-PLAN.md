  <name>Task 1: Create dependency-ordered publish script with dry-run mode</name>
  <files>scripts/publish.sh</files>
  <action>
Create `scripts/publish.sh` following existing script conventions (`set -euo pipefail`, SCRIPT_DIR, REPO_ROOT).

**Script structure:**

1. **Header/usage** — `#!/usr/bin/env bash` with usage comment block: `scripts/publish.sh [--dry-run]`

2. **Setup** — Parse `--dry-run` flag from `$1`. Set `DRY_RUN=true/false`. Define `SCRIPT_DIR`, `REPO_ROOT`.

3. **Crate order array** — Define publish order as an array of crate directory names:
   ```
   CRATES=(
     durable-lambda-core
     durable-lambda-macro
     durable-lambda-closure
     durable-lambda-trait
     durable-lambda-builder
     durable-lambda-testing
   )
   ```
   Core and macro first (no inter-crate runtime deps), then the 4 that depend on core.

4. **Version extraction** — Extract workspace version from root Cargo.toml:
   ```bash
   VERSION=$(grep '^version' "$REPO_ROOT/Cargo.toml" | head -1 | sed 's/.*"\(.*\)"/\1/')
   ```

5. **Already-published detection function** — `is_published(crate_name, version)`:
   - Query crates.io API: `curl -s "https://crates.io/api/v1/crates/$CRATE_NAME/$VERSION"`
   - If HTTP 200 and response contains `"version"`, the version is already published
   - Return 0 (published) or 1 (not published)

6. **Publish loop** — For each crate in CRATES:
   - Print progress: `echo "=== Publishing $CRATE ($i/${#CRATES[@]}) ==="`
   - `cd "$REPO_ROOT/crates/$CRATE"`
   - If `--dry-run`: run `cargo publish --dry-run` and continue to next crate
   - If live publish: check `is_published`. If already published, print skip message and continue. Otherwise run `cargo publish`. On success, wait 30 seconds (with countdown message) for crates.io indexing before next crate. On failure, abort immediately (set -e handles this).

7. **Summary** — Print total crates processed and final status.

**Key behaviors per user decisions:**
- Abort immediately on first failure (set -e is sufficient for this)
- Skip already-published versions on re-run (only in live mode, not dry-run)
- 30-second fixed wait between publishes for crates.io indexing (only in live mode)
- Dry-run does NOT require a crates.io API token
- Exit 0 only if all 6 crates pass

**Claude's discretion choices:**
- Use colored output with green/yellow/red for pass/skip/fail (using ANSI codes, with auto-detection for terminal vs pipe)
- Query crates.io API with curl to detect already-published versions
- Include `--help` flag that prints usage
- Do NOT add `--allow-dirty` support (users should commit before publishing)

Make the script executable: `chmod +x scripts/publish.sh`
