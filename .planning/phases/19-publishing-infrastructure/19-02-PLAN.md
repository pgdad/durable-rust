<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Create crates.io account and store API token</name>
  <action>
The user needs to create a crates.io account and generate an API token. Claude cannot do this — it requires browser-based GitHub OAuth and manual token generation.
  </action>
  <how-to-verify>
Follow these steps in order:

**Step 1: Create crates.io account**
1. Visit https://crates.io in your browser
2. Click "Log in with GitHub" in the top right
3. Authorize crates.io to access your GitHub account
4. Verify your account is created — you should see your GitHub username in the top right

**Step 2: Generate API token**
1. Click your username in the top right → "Account Settings"
2. Or go directly to: https://crates.io/settings/tokens
3. Click "New Token"
4. Name: `durable-rust-publish` (or any descriptive name)
5. Scopes: select "publish-new" and "publish-update" (these are the minimum required)
6. Click "Generate Token"
7. **Copy the token immediately** — it will not be shown again

**Step 3: Store token locally**
Run this command in your terminal (paste your token when prompted):
```bash
cargo login
```
This stores the token in `~/.cargo/credentials.toml`.

**Step 4: Verify**
Run:
```bash
cat ~/.cargo/credentials.toml
```
You should see a line like:
```toml
[registry]
token = "cioXXXXXXXXXXXXXXXXXXXX"
```
  </how-to-verify>
  <resume-signal>Type "done" once you have completed all 4 steps, or describe any issues</resume-signal>
</task>
