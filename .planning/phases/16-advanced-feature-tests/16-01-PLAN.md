**saga_compensation.rs (ADV-01):**
- Register 3 compensable steps via `ctx.step_with_compensation()`: "book_hotel" (returns "HOTEL-001"), "book_flight" (returns "FLIGHT-001"), "charge_card" (returns "CHARGE-001").
- Each compensation closure prints a cancellation message and returns Ok(()).
- Step 4 ("notify_vendor") is a plain `ctx.step()` that always returns `Err::<String, String>("vendor_unavailable".to_string())`.
- When step 4 result is Err, call `ctx.run_compensations().await?` and return `Ok(json!({"status":"rolled_back","compensation_sequence": names_vec, "all_succeeded": comp_result.all_succeeded}))` where names_vec is the Vec of compensation item names from `comp_result.items`.
- The compensation sequence should be LIFO: `["charge_card", "book_flight", "book_hotel"]`.
- Clone any event data before moving into step closures per CLAUDE.md closure patterns.

**step_timeout.rs (ADV-02):**
- Single step "slow_operation" using `ctx.step_with_options()` with `StepOptions::new().timeout_seconds(2)`.
- Closure body: `tokio::time::sleep(Duration::from_secs(60)).await; Ok::<String, String>("done".to_string())`.
- The `?` after `step_with_options` propagates `DurableError::StepTimeout` to the Lambda runtime, causing FunctionError in the invocation response.
- Import `std::time::Duration`.

**conditional_retry.rs (ADV-03):**
- Read `event["error_type"]` as string, default to "non_retryable".
- Single step "call_api" using `ctx.step_with_options()` with `StepOptions::new().retries(3).retry_if(|e: &String| e == "transient")`.
- Closure always returns `Err::<String, String>(err_type)` where err_type is the captured error_type string.
- Clone `error_type` string before moving into the step closure.
- With "transient" error_type: retry_if returns true, `StepRetryScheduled` is propagated on first invoke, execution eventually FAILED after retry budget exhausted.
- With "non_retryable" error_type: retry_if returns false, FAILED immediately (no retries).
- Return `Ok(json!({"result": result}))` after the step (though this line is only reached if step succeeds, which it won't in this test handler).

**batch_checkpoint.rs (ADV-04):**
- Read `event["batch"]` as bool, default false.
- If batch mode: call `ctx.enable_batch_mode()`.
- Run 5 sequential steps named "step_0" through "step_4", each returning `Ok::<i32, String>(i)`.
- If batch mode: call `ctx.flush_batch().await?` after all steps.
- Return `Ok(json!({"steps_completed": 5, "batch_mode": use_batch}))`.
