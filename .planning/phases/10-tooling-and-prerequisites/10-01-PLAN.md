  <action>
Create the `scripts/` directory and `scripts/verify-prerequisites.sh` with the following behavior:

1. **Header and configuration:**
   - `#!/usr/bin/env bash` with `set -euo pipefail`
   - Define constants: `PROFILE="adfs"`, `REGION="us-east-2"`, `ACCOUNT="REDACTED_ACCOUNT_ID"`
   - Define an `ERRORS` counter starting at 0

2. **Tool version display section** ("Tool Versions"):
   - Print version output for: `terraform --version | head -1`, `aws --version`, `docker --version`, `docker buildx version | head -1`, `jq --version`, `rustc --version`
   - Each version command wrapped so a missing tool increments ERRORS and prints `[FAIL] tool-name not found` instead of aborting (use a helper function)

3. **Minimum version checks:**
   - Terraform: parse version number, require >= 1.14.0
   - AWS CLI: require v2.x (parse major version)
   - Docker: require >= 20.0.0
   - jq: require >= 1.7
   - Rust: require >= 1.70.0
   - If any version is below minimum, increment ERRORS and print `[FAIL] tool vX.Y.Z < minimum vA.B.C`

4. **Connectivity checks section** ("Connectivity Checks"):
   - `check "ADFS credentials valid"` runs `aws sts get-caller-identity --profile $PROFILE --region $REGION` -- the `--region us-east-2` flag is critical per user decision (ADFS profile itself is us-east-1, must NOT be modified)
   - `check "ECR accessible (us-east-2)"` runs `aws ecr describe-repositories --profile $PROFILE --region $REGION --max-items 1`
   - `check "Docker daemon responding"` runs `docker info --format '{{.ServerVersion}}'`

5. **Summary and exit:**
   - If ERRORS > 0, print `ERROR: N check(s) failed. Fix before proceeding.` and `exit 1`
   - If ERRORS == 0, print `All prerequisites satisfied.` and `exit 0`

6. **Make executable:** `chmod +x scripts/verify-prerequisites.sh`

Key constraints per user decisions:
- NEVER modify the ADFS profile region -- always use explicit `--region us-east-2` flag
- Use `--profile adfs` on every AWS CLI call
- Docker Desktop uses `desktop-linux` context; check daemon via `docker info`, NOT via systemctl
- Do NOT set `AWS_DEFAULT_REGION` or `AWS_REGION` environment variables
