  <action>
Create `scripts/test-helpers.sh` as a sourceable library of shared functions. This file is NOT executable on its own — it is sourced by test-all.sh. Include these functions in this exact order:

**Header/Constants:**
- `set -euo pipefail` at the top
- `PROFILE="adfs"`, `REGION="us-east-2"` constants
- `SCRIPT_DIR` / `REPO_ROOT` / `TF_DIR` path variables (same pattern as build-images.sh)
- `TERMINAL_STATES=("SUCCEEDED" "FAILED" "TIMED_OUT" "STOPPED")` array
- `TEST_RUN_ID=$(date +%Y%m%d%H%M%S)` for unique execution names

**check_credentials():**
- Calls `aws sts get-caller-identity --profile "$PROFILE" --region "$REGION" --output text`
- On failure: prints "ERROR: ADFS credentials expired or missing. Run: adfs-auth" and calls `exit 1`
- On success: prints "  [OK] ADFS credentials valid"

**load_tf_outputs():**
- Sets global variables: `ALIAS_ARNS`, `STUB_ARNS`, `SUFFIX`
- `ALIAS_ARNS=$(terraform -chdir="$TF_DIR" output -json alias_arns)`
- `STUB_ARNS=$(terraform -chdir="$TF_DIR" output -json stub_alias_arns)`
- `SUFFIX=$(terraform -chdir="$TF_DIR" output -raw suffix)`
- Validates ALIAS_ARNS is non-empty JSON object (not "{}"); exit 1 if empty with message "No alias ARNs found — has terraform apply completed?"

**get_alias_arn(binary_name):**
- `echo "$ALIAS_ARNS" | jq -r --arg name "$1" '.[$name]'`
- Check result is not "null" or empty; if so, echo error to stderr and return 1

**get_stub_arn(stub_name):**
- Same pattern as get_alias_arn but reads from `$STUB_ARNS`
- `echo "$STUB_ARNS" | jq -r --arg name "$1" '.[$name]'`

**make_exec_name(binary_name):**
- `echo "${1}-${TEST_RUN_ID}" | cut -c1-64`

**invoke_sync(function_arn, payload):**
- Creates temp file: `out_file=$(mktemp /tmp/dr-test-XXXXXX.json)`
- Calls `aws lambda invoke --profile "$PROFILE" --region "$REGION" --function-name "$1" --invocation-type RequestResponse --cli-binary-format raw-in-base64-out --payload "$2" "$out_file" 2>&1`
- Captures metadata to local var `meta`
- Extracts: `status_code=$(echo "$meta" | jq -r '.StatusCode // 0')`, `fn_error=$(echo "$meta" | jq -r '.FunctionError // empty')`, `exec_arn=$(echo "$meta" | jq -r '.DurableExecutionArn // empty')`
- Reads response body from `$out_file`, removes temp file
- Outputs pipe-delimited: `echo "$status_code|$fn_error|$exec_arn|$response_body"`

**invoke_async(function_arn, payload):**
- Creates temp file, calls `aws lambda invoke` with `--invocation-type Event`
- Includes `--durable-execution-name "$(make_exec_name "$(basename "$1")")"` for test isolation
- Removes temp file
- Extracts and outputs only: `exec_arn` (DurableExecutionArn from meta JSON)
- If exec_arn is empty, echo error to stderr and return 1

**wait_for_terminal_status(exec_arn, timeout_seconds=120):**
- Default timeout 120 seconds, configurable via `${2:-120}`
- Fixed 3-second polling interval (NOT exponential backoff)
- Polls `aws lambda get-durable-execution --profile "$PROFILE" --region "$REGION" --durable-execution-arn "$1" --query 'Status' --output text`
- Checks if status is in TERMINAL_STATES array
- If terminal: echo status, return 0
- If timeout exceeded: echo "TIMEOUT", return 1
- Uses `sleep 3` between polls (not busy-loop)

**extract_callback_id(exec_arn, timeout_seconds=60):**
- Default timeout 60 seconds
- Fixed 3-second polling interval
- Polls `aws lambda get-durable-execution-history --profile "$PROFILE" --region "$REGION" --durable-execution-arn "$1" --query 'Events[?EventType==\`CallbackStarted\`].CallbackStartedDetails.CallbackId | [0]' --output text`
- If callback_id is non-empty, not "None", not "null": echo it, return 0
- If timeout: echo empty string, return 1

**send_callback_success(callback_id, result_json='{"approved":true}'):**
- Calls `aws lambda send-durable-execution-callback-success --profile "$PROFILE" --region "$REGION" --callback-id "$1" --result "$2" --cli-binary-format raw-in-base64-out`
- Returns the exit code of the aws command

Mark the file as NOT executable (it is sourced, not run directly). Add a header comment: `# scripts/test-helpers.sh — Shared helper functions for Lambda integration tests. Source this file; do not run directly.`

Create `scripts/test-all.sh` as the main test entry point. This file sources test-helpers.sh and provides the test runner framework.

**Header:**
```bash
#!/usr/bin/env bash
# scripts/test-all.sh — Integration test runner for durable-rust Lambda functions.
# Usage:
#   bash scripts/test-all.sh                    # Run all tests
#   bash scripts/test-all.sh closure-basic-steps # Run single named test
#
# Prerequisites: ADFS credentials valid, Lambda functions deployed.
set -euo pipefail
```

**Initialization:**
- `SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"`
- `source "$SCRIPT_DIR/test-helpers.sh"` — loads all helpers
- Result tracking: `PASS_COUNT=0`, `FAIL_COUNT=0`, `SKIP_COUNT=0`, `declare -a FAILURES=()`

**run_test(test_name, test_fn):**
- Printf-formats test name left-aligned in 55 chars
- Runs `$test_fn` in a subshell capturing stdout+stderr: `if error_msg=$("$test_fn" 2>&1); then`
- On success: prints `[PASS]`, increments PASS_COUNT
- On failure: prints `[FAIL]`, appends `"$test_name: $(echo "$error_msg" | tail -1)"` to FAILURES array, increments FAIL_COUNT
- Critical: the `if` conditional prevents `set -e` from aborting on test failure

**print_results():**
- Prints blank line then summary: `=== Results: N passed, M failed, K skipped ===`
- If failures exist, prints each: `  FAIL: {test_name}: {reason}`
- Returns exit code: `[[ $FAIL_COUNT -eq 0 ]]`

**Stub test functions (Phase 14-16 will replace these with real implementations):**

Create one stub function per binary name. Each stub should be a function that returns 0 (pass) with a comment `# TODO: Phase {14|15|16} will implement`.

Group the stubs by test type with comments:

```bash
# === Phase 14: Synchronous Operation Tests ===

#  replay_safe_logging, combined_workflow — 4 styles each = 32 tests)

test_macro_basic_steps() { echo "STUB — not yet implemented"; }
test_trait_basic_steps() { echo "STUB — not yet implemented"; }

#     child_contexts, replay_safe_logging, combined_workflow

# (waits, callbacks, invoke — 4 styles each = 12 tests)

# ... etc

# === Phase 16: Advanced Feature Tests ===
# (saga_compensation, step_timeout, conditional_retry, batch_checkpoint)

```

Use function names matching binary names with `test_` prefix and hyphens converted to underscores: `closure-basic-steps` becomes `test_closure_basic_steps`.

**BINARY_TO_TEST associative array:**
- Maps binary names to test function names for single-test dispatch
- Example: `BINARY_TO_TEST["closure-basic-steps"]="test_closure_basic_steps"`
- Include all 44 entries

**run_all_tests():**
- Calls `run_test` for each test in defined order (Phase 14 sync tests first, then Phase 15 async, then Phase 16 advanced)
- Uses descriptive test name as first arg: e.g., `run_test "closure-basic-steps" test_closure_basic_steps`

**main():**
- Prints banner: `=== durable-rust Integration Tests ===`
- Calls `check_credentials` (from test-helpers.sh) — TEST-04 credential gate
- Calls `load_tf_outputs` — read Terraform outputs
- If `$1` is provided: look up in BINARY_TO_TEST map; if found, call `run_test "$1" "${BINARY_TO_TEST[$1]}"`. If not found, print "Unknown test: $1" with available test names and exit 1.
- If no `$1`: call `run_all_tests`
- Call `print_results`
- Exit with print_results return code (0 if all pass, 1 if any fail)

`main "$@"` at the bottom.

Make the file executable: `chmod +x scripts/test-all.sh`
