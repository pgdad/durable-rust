**TEST-12: Zero-duration wait**

1. `test_zero_duration_wait_execute_path` — Use `MockDurableContext::new().build().await` (no pre-loaded ops = execute mode). Call `ctx.wait("zero_wait", 0).await`. Assert the result is `Err(e)` where `e` is `DurableError::WaitSuspended`. This proves zero-duration wait is valid and behaves like any other wait on execute path.

2. `test_zero_duration_wait_replay_path` — Use `MockDurableContext::new().with_wait("zero_wait").build().await` (pre-loaded wait = replay mode). Call `ctx.wait("zero_wait", 0).await`. Assert the result is `Ok(())`. This proves zero-duration wait replays correctly. Verify no checkpoints were made: `assert_no_checkpoints(&calls).await`.

**TEST-13: Map batch_size edge cases**

3. `test_map_batch_size_zero_panics` — Use `#[should_panic(expected = "MapOptions::batch_size: size must be > 0, got 0")]`. Call `MapOptions::new().batch_size(0)`. This confirms the Phase 4 validation at integration level.

4. `test_map_batch_size_one_processes_sequentially` — Use `MockDurableContext::new().build().await`. Create a 3-item Vec `vec![1, 2, 3]`. Call `ctx.map("batch1", items, |item: i32, _ctx: DurableContext| async move { Ok(item * 10) }, MapOptions::new().batch_size(1)).await.unwrap()`. Assert `result.results.len() == 3`. Assert each item's `status` is `BatchItemStatus::Succeeded` and values are 10, 20, 30 (verify by checking `result.results` sorted by `index`).

5. `test_map_batch_size_exceeds_collection` — Use `MockDurableContext::new().build().await`. Create a 2-item Vec `vec![1, 2]`. Call `ctx.map("big_batch", items, |item: i32, _ctx: DurableContext| async move { Ok(item * 5) }, MapOptions::new().batch_size(100)).await.unwrap()`. Assert `result.results.len() == 2` with values 5 and 10. This proves batch_size > collection is handled gracefully.

**TEST-14: Parallel with 0 and 1 branches**

6. `test_parallel_zero_branches` — Use `MockDurableContext::new().build().await`. Declare a type alias for the branch function type:
```rust
type BranchFn = Box<dyn FnOnce(DurableContext) -> Pin<Box<dyn Future<Output = Result<i32, DurableError>> + Send>> + Send>;
```
Create empty `Vec::<BranchFn>::new()`. Call `ctx.parallel("empty", branches, ParallelOptions::new()).await.unwrap()`. Assert `result.results.len() == 0`. Verify exactly 2 checkpoints (outer START + SUCCEED): `let captured = calls.lock().await; assert_eq!(captured.len(), 2)`.

7. `test_parallel_one_branch` — Use `MockDurableContext::new().build().await`. Create a single-branch vector with a closure returning `Ok(42i32)`. Call `ctx.parallel("single", branches, ParallelOptions::new()).await.unwrap()`. Assert `result.results.len() == 1` and the single item has status `Succeeded` with result `Some(42)`.

**TEST-15: Operation name edge cases**

8. `test_operation_name_empty_string` — Use `MockDurableContext::new().build().await`. Call `ctx.step("", || async { Ok(1i32) }).await.unwrap()`. Assert result is `Ok(1)`. Verify operation was recorded with `assert_operations(&ops, &["step:"]).await`.

9. `test_operation_name_unicode` — Use `MockDurableContext::new().build().await`. Call `ctx.step("こんにちは世界", || async { Ok(1i32) }).await.unwrap()`. Assert result is `Ok(1)`. Verify with `assert_operations(&ops, &["step:こんにちは世界"]).await`.

10. `test_operation_name_long_255_plus_chars` — Use `MockDurableContext::new().build().await`. Let `long_name = "a".repeat(300)`. Call `ctx.step(&long_name, || async { Ok(1i32) }).await.unwrap()`. Assert result is `Ok(1)`. Verify operation name was recorded: read `ops.lock().await` and assert the first operation's name equals `long_name`.

**TEST-16: Negative option values (integration-level confirmation)**

**Important notes:**

- For parallel branch closures, the closure receives `DurableContext` (not `&mut DurableContext`).
