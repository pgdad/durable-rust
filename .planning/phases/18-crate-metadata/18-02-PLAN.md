From crates/durable-lambda-core:
- DurableContext: main context type with step, wait, callback, invoke, parallel, map, child_context, log methods
- DurableError: error type for SDK infrastructure errors
- StepOptions: retries, backoff, timeout, retry_if
- ParallelOptions, MapOptions, CallbackOptions
- DurableBackend trait: RealBackend (AWS) + MockBackend (testing)
- Replay engine: HashMap<String, Operation>, blake2b operation IDs

From crates/durable-lambda-macro:
- #[durable_execution] attribute macro on async fn
- Validates second param is DurableContext, return is Result<_, DurableError>
- Generates main() with Lambda runtime boilerplate

From crates/durable-lambda-closure:
- ClosureContext wrapping DurableContext
- durable_lambda_closure::run(handler_fn) entry point
- prelude module re-exporting all types

From crates/durable-lambda-trait:
- DurableHandler trait with async fn handle(&self, event, ctx)
- TraitContext wrapping DurableContext
- durable_lambda_trait::run(MyHandler) entry point
- prelude module re-exporting all types

From crates/durable-lambda-builder:
- DurableHandlerBuilder with .with_tracing(), .with_error_handler()
- BuilderContext wrapping DurableContext
- durable_lambda_builder::handler(closure).run() entry point
- prelude module re-exporting all types

From crates/durable-lambda-testing:
- MockDurableContext::new() builder
- .with_step_result(), .with_step_error(), .with_wait(), .with_callback(), .with_invoke()
- .build() -> (DurableContext, CheckpointRecorder, OperationRecorder)
- assert_no_checkpoints(), assert_checkpoint_count(), assert_operations(), assert_operation_names(), assert_operation_count()
