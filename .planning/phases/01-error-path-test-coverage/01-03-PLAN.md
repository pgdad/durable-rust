Modify `step_with_options` in `crates/durable-lambda-core/src/operations/step.rs` to catch panics in the step closure.

**The change:** Replace line 207 (`let user_result = f().await;`) with a `tokio::spawn` wrapper that catches panics as `JoinError`:

```rust
// Execute the closure in a spawned task to catch panics.
let name_owned = name.to_string();
let handle = tokio::spawn(async move { f().await });
let user_result = handle.await.map_err(|join_err| {
    DurableError::checkpoint_failed(
        &name_owned,
        std::io::Error::new(
            std::io::ErrorKind::Other,
            format!("step closure panicked: {join_err}"),
        ),
    )
})?;
```

**Signature change required:** Because `tokio::spawn` requires `'static`, the closure bounds must change. Update BOTH `step` and `step_with_options` signatures in step.rs:

From:
```rust
F: FnOnce() -> Fut + Send,
Fut: Future<Output = Result<T, E>> + Send,
```

To:
```rust
F: FnOnce() -> Fut + Send + 'static,
Fut: Future<Output = Result<T, E>> + Send + 'static,
```

Also add `'static` to T and E bounds:
```rust
T: Serialize + DeserializeOwned + Send + 'static,
E: Serialize + DeserializeOwned + Send + 'static,
```

**Why this is safe:** Step closures already require `Send` (same as parallel branches). Users already clone data before moving into step closures (documented pattern in CLAUDE.md: `ctx.step("name", || { let e = event.clone(); async move { ... } })`). The `'static` bound is practically always satisfied because closures move owned data. This aligns step with the same safety model as parallel branches.

**Propagation to wrapper crates (REQUIRED):** All three wrapper crates re-declare step bounds independently. Update each to match the new core bounds:

1. `crates/durable-lambda-closure/src/context.rs` — Update bounds on `step` (lines 90-93) and `step_with_options` (lines 137-140): add `+ 'static` to T, E, F, and Fut bounds.

2. `crates/durable-lambda-trait/src/context.rs` — Update bounds on `step` (lines 100-103) and `step_with_options` (lines 147-150): add `+ 'static` to T, E, F, and Fut bounds.

3. `crates/durable-lambda-builder/src/context.rs` — Update bounds on `step` (lines 96-99) and `step_with_options` (lines 143-146): add `+ 'static` to T, E, F, and Fut bounds.

Each wrapper has 2 methods x 4 bounds = 8 bound lines to update per crate, 24 total across all three wrappers. The wrappers are thin pass-throughs to `self.inner.step(...)` so no logic changes are needed — only bounds.

```rust
async fn test_step_closure_panic_returns_error() {
    let (mut ctx, _calls, _ops) = MockDurableContext::new().build().await;

    let result: Result<Result<i32, String>, DurableError> = ctx
        .step("panicking_step", || async {
            panic!("deliberate panic in step closure");
        })
        .await;

    // Should return Err(DurableError::CheckpointFailed) not crash the process
    let err = result.expect_err("step with panicking closure should return Err");
    assert!(
        matches!(err, DurableError::CheckpointFailed { .. }),
        "expected CheckpointFailed from panic, got: {err:?}"
    );
    // Verify the error message mentions the panic
    let msg = err.to_string();
    assert!(
        msg.contains("panicked"),
        "error message should mention panic: {msg}"
    );
}
```
