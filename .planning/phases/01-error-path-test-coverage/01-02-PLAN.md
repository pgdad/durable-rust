1. **TEST-08: test_parallel_all_branches_fail** — Use `MockDurableContext::new().build().await` to get a context in execute mode. Create branches where all return `Err(DurableError::parallel_failed(...))`:
```rust
let branches: Vec<Box<dyn FnOnce(DurableContext) -> Pin<Box<dyn Future<Output = Result<i32, DurableError>> + Send>> + Send>> = vec![
    Box::new(|_ctx| Box::pin(async move {
        Err(DurableError::parallel_failed("b0", "branch 0 failed"))
    })),
    Box::new(|_ctx| Box::pin(async move {
        Err(DurableError::parallel_failed("b1", "branch 1 failed"))
    })),
];
```
Call `ctx.parallel("all_fail", branches, ParallelOptions::new()).await`. Assert the result is `Ok(batch_result)` (NOT Err). Assert `batch_result.results.len() == 2`. Assert ALL items have `BatchItemStatus::Failed`. Assert ALL items have `error.is_some()`. This confirms branch-level failures are captured in BatchResult, not propagated as SDK errors.

2. **TEST-09: test_map_item_failures_at_different_positions** — Use `MockDurableContext::new().build().await`. Create a 5-item map where items at index 0 (first), 2 (middle), and 4 (last) return errors:
```rust
let items = vec![0i32, 1, 2, 3, 4];
let result = ctx.map("position_test", items, MapOptions::new(), |item: i32, _ctx: DurableContext| async move {
    if item == 0 || item == 2 || item == 4 {
        Err(DurableError::map_failed("item", format!("item {item} failed")))
    } else {
        Ok(item * 10)
    }
}).await.unwrap();
```
Assert `result.results.len() == 5`. Assert items [0], [2], [4] have `BatchItemStatus::Failed` and `error.is_some()`. Assert items [1], [3] have `BatchItemStatus::Succeeded` and `result == Some(10)` / `Some(30)`. This confirms per-item error isolation at all positions.

**Important:** Read `map.rs` source before writing this test to confirm the exact closure signature and how map captures per-item errors. The map operation spawns items with `tokio::spawn` similar to parallel, so item-level errors should be captured in BatchResult.

3. **TEST-11: test_parallel_branch_panic_returns_error** — Use `MockDurableContext::new().build().await`. Create branches where one panics:

let branches: Vec<BranchFn> = vec![

let result = ctx.parallel("panic_test", branches, ParallelOptions::new()).await;

Assert the result is `Err(DurableError::ParallelFailed { .. })` (because `handle.await` on a panicked task returns `JoinError`, which the `?` operator propagates as `DurableError::ParallelFailed`).
