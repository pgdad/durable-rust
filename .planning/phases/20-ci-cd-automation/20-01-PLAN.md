<task type="auto">
  <name>Task 1: Create release workflow for tag-triggered publishing</name>
  <files>.github/workflows/release.yml</files>
  <action>
Create `.github/workflows/release.yml` with the following structure:

**Trigger:** `on: push: tags: ['v*']`

**Job 1: test** (runs full validation before publishing)
- `runs-on: ubuntu-latest`
- Steps: checkout@v4, rust-toolchain@stable (with rustfmt, clippy components), cache@v4 (same key pattern as ci.yml)
- Run: `cargo fmt --all --check`
- Run: `cargo clippy --workspace -- -D warnings`
- Run: `cargo test --workspace`

**Job 2: publish** (publishes crates to crates.io)
- `needs: test` (only runs after tests pass)
- `runs-on: ubuntu-latest`
- `permissions: contents: write` (needed for GitHub Release creation)
- Steps:
  - checkout@v4
  - rust-toolchain@stable
  - cache@v4 (same key pattern)
  - Set CARGO_REGISTRY_TOKEN from secrets: `env: CARGO_REGISTRY_TOKEN: ${{ secrets.CARGO_REGISTRY_TOKEN }}`
  - Run: `bash scripts/publish.sh` (live publish, not dry-run)
  - Create GitHub Release using `softprops/action-gh-release@v2` with `generate_release_notes: true`. Use `${{ github.ref_name }}` as the release name/tag.

**Environment variables:**
- Top-level `env: CARGO_TERM_COLOR: always` (matches ci.yml pattern)

Use the scripts/publish.sh script for publishing per user decision (reuse existing script, single source of truth). The script handles dependency ordering and crates.io indexing waits.
