  <action>
Create `crates/durable-lambda-core/src/ops_trait.rs` containing:

1. Module doc comment (`//!`) explaining the trait is the shared interface for all context types.

2. The `DurableContextOps` trait with all 21+ public methods from DurableContext, using **native async fn in traits** (RPITIT) -- NOT `#[async_trait]`. The trait is for static dispatch only (never `dyn`), so no boxing needed. Use `-> impl Future<Output = ...> + Send` return syntax for async methods to match the existing method signatures exactly.

3. Copy the exact generic bounds from the existing DurableContext methods listed in the interfaces block above. Critical: `parallel` and `map` closures take `crate::context::DurableContext` as parameter, NOT `impl DurableContextOps`. This is by design -- child contexts are always the core type.

4. Include rustdoc with `# Examples` section on the trait showing a generic function `async fn logic<C: DurableContextOps>(ctx: &mut C)`.

5. `impl DurableContextOps for DurableContext` in the same file, directly below the trait. Each method delegates to the inherent method: `DurableContext::step(self, name, f)` or just `self.step(name, f)` (the compiler will resolve the inherent method since trait methods use UFCS).

For the DurableContext impl, the async methods should use this pattern:
```rust
fn step<T, E, F, Fut>(&mut self, name: &str, f: F) -> impl Future<Output = Result<Result<T, E>, DurableError>> + Send
where ...
{
    DurableContext::step(self, name, f)
}
```

The sync methods delegate directly:
```rust
fn execution_mode(&self) -> ExecutionMode {
    DurableContext::execution_mode(self)
}
```

6. In `crates/durable-lambda-core/src/lib.rs`, add `pub mod ops_trait;` and `pub use ops_trait::DurableContextOps;` to make the trait publicly accessible.

IMPORTANT: Do NOT remove any existing inherent methods from DurableContext. The trait impl and inherent methods coexist. Rust prefers inherent methods for direct calls; the trait is for generic bounds only. Do NOT use `#[async_trait]` on this trait.
