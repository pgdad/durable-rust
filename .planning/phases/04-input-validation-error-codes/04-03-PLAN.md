<objective>
Replace all 13 silent `if let Some(token)` checkpoint token handling sites with defensive `.ok_or_else(...)` error propagation, so a missing checkpoint_token in AWS responses surfaces as a typed DurableError instead of silently continuing with a stale token.

Purpose: A missing checkpoint_token after a checkpoint write indicates an AWS API contract violation. Silent continuation with a stale token causes subtle downstream corruption. Surfacing it as DurableError::checkpoint_failed makes the failure visible and debuggable.
Output: All 7 operation files updated with defensive checkpoint token handling across 13 sites.
</objective>
