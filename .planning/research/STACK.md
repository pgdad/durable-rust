### `_bmad/` — 508 files, 5 subdirectories

BMAD tooling framework installed into the project repository. Contains the full BMAD agent system, workflows, templates, and configuration. This is tooling infrastructure, not project artifacts.

```
_bmad/
├── _config/        # BMAD agent manifests, IDE configs, skill/tool/workflow manifests
├── _memory/        # BMAD runtime memory (config.yaml, tech-writer-sidecar/)
├── bmm/            # BMAD module: agents (analyst, architect, dev, pm, qa, sm, tech-writer, ux-designer)

└── tea/            # BMAD tea module
```

**Disposition:** Full directory removal. No content from `_bmad/` maps to GSD — GSD tooling lives at `~/.claude/get-shit-done/`, not in the project repository.

## What NOT to Use

| Avoid | Why | Use Instead |
|-------|-----|-------------|
| Migrating `_bmad-output/` content into `.planning/` | BMAD story/epic format doesn't map to GSD phase/plan format; creates noise without value | Capture summary in `MILESTONES.md` v1.0 entry; full history lives in git |
| Keeping `_bmad/` for reference | 508 files of framework tooling with no runtime value in a GSD workflow | Delete entirely; GSD tooling is at `~/.claude/get-shit-done/` |
| Expanding `config.json` unnecessarily | Minimal config is valid; GSD uses defaults | Leave minimal config unless specific behavior overrides are needed |
